# レビューチェックリスト

やらかしリストから逆引きしたチェックリスト。
このプロジェクト（hitoplp）に該当する項目のみ有効化済み。

---

## 必須チェック項目

### ARCH-001: 300行制限
- [ ] 全 `.ts` / `.tsx` ファイルが300行以下か
- 検出: `wc -l` で全ファイルチェック
- 違反時: ファイル分割が完了するまで次に進まない

### ARCH-002: コンポーネントの責務過多
- [ ] 1コンポーネントが1責務に収まっているか
- 検出: コードリーディングで判断

### LOGIC-001: タイムゾーン処理
- [ ] `new Date()` がタイムゾーン安全コメントなしで使われていないか
- 検出: `grep -rn "new Date()" apps/ --include="*.ts" --include="*.tsx"`
- 代替: タイムゾーンを明示するか `// timezone-safe` コメントを付与

### LOGIC-002: エッジケース考慮
- [ ] API呼び出しのエラーハンドリング（fal.ai, Anthropic, ElevenLabs）
- [ ] ネットワークエラー・タイムアウトの処理
- [ ] ユーザー入力のバリデーション

### SEC-001: セキュリティ基本チェック
- [ ] クライアントコード（apps/web/src/）にシークレットがハードコードされていないか
- [ ] APIキーは環境変数経由で取得しているか
- 検出: `grep -rn "sk-\|secret_key\|PRIVATE_KEY" apps/web/src/`

### エラーハンドリング
- [ ] 空 catch ブロックがないか
- 検出: `grep -rn "catch\s*(.*)\s*{\s*}" apps/`

### console.log
- [ ] 本番コードに console.log が残っていないか（console.error, console.warn は許可）
- 検出: `grep -rn "console\.log" apps/web/src/`
